# 기출문제 PDF → 시험 대비 flashcard JSON 변환 가이드

전공서용 `system_prompt.md`의 **자매 문서**다. 목적만 다르다: 전공서 한 챕터를 심층 학습 흐름으로
재구성하는 대신, **시험 기출문제 뭉치를 "문제 + 풀이·해설" flashcard 덱**으로 만든다. 출력 JSON의
스키마·이스케이프·시각화 규칙은 `system_prompt.md`와 **완전히 동일**하다(그 문서를 먼저 읽어라).
여기서는 그 위에 얹는 *시험 대비 전용 규칙*만 정리한다.

이 콘텐츠는 **임시**다 — `data/exam/`에만 저장되고, 시험이 끝나면 통째로 지운다(아래 「등록·삭제」).
전공서 학습 데이터(`data/*.json`)와는 폴더부터 분리돼 있어 서로 절대 영향을 주지 않는다.

---

## 입력 (사용자가 준다)

1. **기출문제 형식의 PDF 뭉치** — 한 과목의 여러 회차/여러 문제.
2. **그때그때 다른 추가 요구사항** — 예: "풀이는 단계별로 아주 자세히", "각 문제에 난이도·출제의도 한 줄",
   "영어 문제는 한국어로 번역해 함께", "O/X 복습은 넣지 말고 서술형 풀이만", "특정 챕터만" 등.
   **이 추가 요구사항이 아래 기본 규칙과 충돌하면, 언제나 사용자의 추가 요구사항이 우선한다.**

## 출력 (이 가이드가 만드는 것)

기존 카드 덱 엔진이 그대로 렌더링하는, **전공서와 똑같은 스키마의 JSON 하나**.
`chapter_info` + `learning_flow[]`. 시험용에서는 각 요소를 이렇게 대응시킨다.

### `chapter_info`
- `title`: 이 시험 세트의 이름. 예: `"2024-2 미적분학 중간고사 대비"`. (드롭다운에 뜨는 이름은
  등록 시 `--title`로 덮어쓸 수 있으니, 여기서는 자연스러운 제목이면 된다.)
- `source`: 출처. 예: `"KAIST MAS101 2022~2024 기출"`.
- `overall_goal`: 이 세트로 무엇을 대비하는지 한 줄.

### `learning_flow[]` — 섹션 = "문제 묶음" 하나
기출을 그냥 문제 번호대로 죽 나열하지 말고, **주제(또는 회차)별로 묶어** 한 묶음을 한 섹션으로 만든다.
예: "극한·연속", "급수 수렴판정", "2023 기말 1~4번". 사용자가 묶는 기준을 지정하면 그대로 따른다.

각 섹션:
- `section_title`: 그 묶음 이름. 예: `"II. 급수의 수렴 판정"`.
- `step1_big_picture` (**필수** — 없으면 검증 오류):
  - `context`: 이 유형이 시험에서 어떻게/왜 나오는지, 무엇을 묻는지 한두 문장.
  - `hidden_intuition`: 이 유형을 푸는 **핵심 전략·판단 기준**(어떤 판정법을 언제 쓰는가, 흔한 함정 등).
    시험 대비에서 가장 값어치 있는 칸이니 형식적으로 얼버무리지 마라.
- `step2_rigorous_logic` (**선택** — 비워도 됨, 비면 경고만 뜨고 저장은 된다):
  - 그 유형을 풀 때 **반복적으로 쓰이는 정의·정리·공식**을 `Definition`/`Theorem` 등 블록으로 몇 개 정리해
    두면 복습에 좋다(예: 비판정법 진술). 순수 문제풀이 덱만 원하면 `[]`로 둔다.
- `step3_checkpoint[]` (**핵심** — 여기에 기출문제가 들어간다):
  - 각 기출문제 → `type: "checkpoint"` 항목 하나:
    `{ "type":"checkpoint", "problem":"...", "hint":"...", "solution":"...", "visualization"?:{...} }`
    - `problem`: 기출문제를 **정확히** 옮긴다(필요하면 자연스러운 한국어로 다듬되 뜻을 바꾸지 마라).
      선택지가 있으면 함께. 원문이 영어면 사용자 요구에 따라 번역/병기.
    - `hint`: 답을 말하지 않되 접근 방향을 여는 실마리.
    - `solution`: **단계가 보이는 상세 풀이 + 최종 답.** 시험 대비의 본체다. 사용자가 "간단히"라고 하면
      줄이고, "아주 자세히"라고 하면 모든 변형 단계를 남긴다.
  - 원하면 그 묶음 개념의 경계를 짚는 `type:"ox"` / `type:"conceptual"` 항목을 덧붙일 수 있다(기본 스키마
    그대로). **단, 시험 대비에서는 억지로 만들지 말 것** — 사용자가 요청할 때나, 흔한 실수를 경고할 가치가
    분명할 때만. 배열 순서는 `checkpoint`들 → `ox`들 → `conceptual` 순.

## 반드시 지킬 것 (system_prompt.md와 공통)
- 출력은 **유효한 JSON 객체 하나만**. 코드펜스·설명·주석 금지. 첫 글자 `{`, 끝 글자 `}`.
- **모든 LaTeX 백슬래시는 `\\`로 이스케이프**(예: `"$\\int_0^1 x^2\\,dx$"`). 집합 괄호도 `\\{ \\}`.
- 서술은 한국어, 표준 용어는 한/영 병기. 모든 수식은 KaTeX(`$...$` / `$$...$$`).
- 그림이 필요하면 `system_prompt.md`의 `visualization` 화이트리스트를 그대로 쓴다(`checkpoint` 항목에만).
- 검증기(`lib/merger.py`)는 전공서와 동일하게 돈다: `problem` 없는 checkpoint, `answer`가 O/X가 아닌 ox
  등은 **오류(✗)**라 저장이 막힌다. `step2` 비었음·`ox` 없음 등은 **경고(⚠)**일 뿐 통과한다.

## 등록·삭제 (전공서 파이프라인과 별개)
생성한 JSON은 `pipeline/exam.py`로만 다룬다(수동으로 `data/manifest.json`을 건드리지 않는다):
```
python pipeline/exam.py add --json <파일> [--title "드롭다운에 뜰 이름"] [--book "그룹명"]
python pipeline/exam.py list                     # 현재 시험 세트 목록
python pipeline/exam.py remove --id <id>         # 한 세트만 삭제
python pipeline/exam.py clear --yes              # 시험 세트 전체 비우기(전공서엔 영향 없음)
```
`add`가 검증 → `data/exam/`에 저장 → `data/exam/manifest.json` 등록까지 한다. 등록된 세트는 사이트
주제 드롭다운의 별도 그룹(`--book`, 기본 `📝 시험 대비`)에 나타난다. 시험이 끝나면 `clear`로 통째로 비우고
커밋하면 된다.
